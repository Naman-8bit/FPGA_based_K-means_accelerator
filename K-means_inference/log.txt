1. The Core Problem: Serial vs. Parallel
Inside your Tang Nano 9K, data is Parallel. If you have an 8-bit wire, all 8 bits travel side-by-side simultaneously. It is incredibly fast, but it requires 8 physical pins and 8 physical wires.

Your USB cable only has one dedicated data wire for transmitting (TX) and one for receiving (RX). You cannot send 8 bits at the same time over one wire. You have to send them sequentially: one bit after another. This is called Serial communication.

A UART (Universal Asynchronous Receiver-Transmitter) is the translation machine. It takes an 8-bit parallel chunk, loads it into a shift register, and pushes it out a single pin, one bit at a time.


2. What is Baud Rate? (The "Asynchronous" Secret)
The "A" in UART stands for Asynchronous. This means there is no shared clock wire between your computer and your FPGA.

If your FPGA sends a 1 over the wire, how does your computer know if that is a single 1, or three 1s sent back-to-back? The only way this works is if both sides agree on a strict stopwatch before the conversation begins.

This stopwatch is the Baud Rate: the exact number of bits transmitted per second.
* Standard rates are 9600, 115200, 921600, etc.
* Is it changeable? Absolutely. You can make it whatever you want, as long as both the Python script and the FPGA are configured to the exact same number. We use 115,200 because it is the standard maximum speed for reliable FTDI USB connections without data corruption.

To code this, you calculate a "Clock Divider":
Clocks_Per_Bit = System_Clock_Frequency / Baud_Rate

For your board: 27,000,000 / 115,200 = 234. Your FPGA must wait exactly 234 clock ticks to read or write a single bit.


3. The UART Data Frame
Because the line is asynchronous, the receiver has to know exactly when a new byte is starting, otherwise it would just read random electrical noise. A standard UART frame always looks like this:
* Idle State: The wire is held HIGH (1).
* The Start Bit: The transmitter pulls the wire LOW (0) for one full bit-duration. This is the "wake up" signal.
* The Data Bits: The 8 bits of your byte are sent, usually Least Significant Bit (LSB) first.
* The Stop Bit: The transmitter pulls the wire HIGH (1) for at least one full bit-duration to signal the end of the byte.
