### Bytes/Pixel In — Standard
- **Verilog FSM:** `K-means_inference/rtl/inference.v`, lines 50–76 — states `Wait_R`, `Wait_G`, `Wait_B` receive 3 sequential UART bytes (R, G, B).

### Bytes/Pixel In — BRAM
- **Verilog FSM:** `K-means_inference/rtl/inference_bram_v2.v`, lines 172–193 — states `REC_R`, `REC_G`, `REC_B` receive 3 sequential UART bytes per pixel, packed as `px_din <= {R, G, rx_byte}` (24-bit).
- **Python:** `K-means_inference/python scripts/host_bram.py`, lines 43–58 — pixels are reshaped to (N, 3), then `chunk.tobytes()` sends 3 bytes per pixel.

### Bytes/Pixel Out — Standard
- **Verilog FSM:** `K-means_inference/rtl/inference.v`, line 82: `tx_byte<={6'd0,winning_cluster}` — sends 1 byte (cluster ID in bottom 2 bits).
- **Python:** `K-means_inference/python scripts/host.py`, line 47: `response = ser.read(1)` — reads 1 byte per pixel.

### Bytes/Pixel Out — BRAM
- **Verilog FSM:** `K-means_inference/rtl/inference_bram_v2.v`, line 215: `res_din <= {6'b000000, winning_cluster}` — stores 1 byte per pixel result; line 243: `tx_byte <= res_dout` — transmits 1 byte.
- **Python:** `K-means_inference/python scripts/host_bram.py`, line 62: `result_bytes = ser.read(valid_pixels)` — reads 1 byte per pixel.

### Batch Size — BRAM
- **Verilog:** `K-means_inference/rtl/inference_bram_v2.v`, line 20: `parameter DEPTH = 8192`
- **Python:** `K-means_inference/python scripts/host_bram.py`, line 10: `CHUNK_SIZE  = 8192`
- The host sends an `OP_LOAD` (0xAA) opcode followed by a 2-byte big-endian pixel count (`struct.pack('>H', valid_pixels)`, line 53), then streams `valid_pixels × 3` data bytes per chunk.

### Batch Size — Standard
- Not applicable. The standard variant uses a single-pixel handshake protocol (send 3 bytes → receive 1 byte → repeat).
